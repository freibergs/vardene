vardene

Latvian morphological analysis — Python port of the LU MII engine

Analyse a single word

Returns every candidate reading the morphology engine produces for a word, with full attribute set. Maps to GET /api/analyze/<word> and /api/analyze/en/<word>.

Analyse a sentence

Two modes — Best reading uses sentence-level CRF + Viterbi disambiguation (/api/morphotagger); All readings shows every candidate per token (/api/analyzesentence).

Tokenise text

FSA-driven tokeniser ported from Splitting.java. Recognises clocks (14:30), URLs, e-mails, ISO dates, paragraph numbers (1.2.3.4.), and more. Maps to GET /api/tokenize/<query>.

Inflect a lemma

All inflected forms of a lemma. Optional fields: paradigm name to force a specific paradigm; for verb-1 conjugation you can pass three explicit stems. Language filter restricts forms to Latvian (lv) or Latgalian (ltg). Maps to /api/v1/inflections and /api/inflect/json.

Advanced options

Suitable paradigms

For a lemma, list every paradigm that could generate it, sorted by ending frequency (most common first). Maps to GET /api/suitable_paradigm/<lemma>.

Inflect a phrase

Multi-word noun-phrase declension. The rightmost noun is the head; modifiers (adjectives, pronouns) agree in case + number + gender. Normalise brings any phrase form back to its lemma. Maps to /api/inflect_phrase and /api/normalize_phrase.

Inflect a personal name

Each component (given name + surname) is inflected through all 6 cases × 2 numbers. Setting gender filters readings to that gender; if no matching reading exists, the name is returned as-is. Maps to GET /api/inflect_people/json/<name>.

Valency-tag annotation

For a word or short phrase, returns the list of valency-relevant tags it could attach to (case codes for nouns, V1/V2/V3/Inf for verbs, Adv for adverbs, S for subordinate-clause heads, etc.). Used by the upstream verb-valency annotation tool. Verb mode biases the reading toward verb POS; non-verb mode biases away from it. Maps to /api/verbs and /api/neverbs.

HTTP API reference

Full parity with api.tezaurs.lv 2.5.15. All endpoints return JSON encoded as raw UTF-8.
